## auth.ts

Exports `auth` ‚Äî the better-auth instance, and `requireSession` ‚Äî a shared auth guard.

**Config:**
- Database: Drizzle adapter with PostgreSQL
- Provider: email/password with sign-up limit (100 users max via `beforeSignUp` hook)
- Features:
  - **Email verification** ‚Äî `requireEmailVerification: true` forces users to verify email before login
  - **Forgot password** ‚Äî built-in password reset flow with `sendResetPassword` callback
  - **Email callbacks** ‚Äî `sendVerificationEmail` and `sendResetPassword` handle email templates
  - **Password reset hook** ‚Äî `onPasswordReset` callback fires after successful reset
- Plugins:
  - `sveltekitCookies(getRequestEvent)` ‚Äî handles cookie injection per request

**Usage in API routes:**
```ts
import { requireSession } from '$lib/server/auth';

const session = await requireSession(request.headers);
// session.user.id, session.user.email
// throws 401 automatically if no session
```

**Sign-up Limit:**
- Max 100 users enforced via `beforeSignUp` hook
- Attempting sign-up when limit is reached returns error code `USER_LIMIT_REACHED`

**Email Sending:**
- `sendVerificationEmail` ‚Äî called when user signs up or requests email verification; currently logs to console (TODO: implement Resend, SendGrid, or Nodemailer)
- `sendResetPassword` ‚Äî called when user requests password reset; currently logs to console (TODO: implement actual email service)
- Both functions receive user data and verification/reset URL; avoid awaiting email sending to prevent timing attacks

## db/index.ts

Exports `db` ‚Äî Drizzle ORM client connected to PostgreSQL via `postgres-js` driver.

Connection string from `DATABASE_URL` env var.

**Usage:**
```ts
import { db } from '$lib/server/db';
import { paper } from '$lib/server/db/schema';
import { eq } from 'drizzle-orm';

// Prefer relational queries to avoid N+1 problems
const rows = await db.query.paper.findMany({
  where: eq(paper.userId, userId),
  with: { references: true }  // Load related references in one query
});

// For simple single-row lookups
const row = await db.query.paper.findFirst({
  where: eq(paper.id, id)
});
```

## db/schema.ts

Custom Drizzle schema. Re-exports `auth.schema.ts`.

**Tables:**

`paper`
| Column | Type | Notes |
|---|---|---|
| `id` | text PK | `crypto.randomUUID()` |
| `userId` | text FK ‚Üí user | cascade delete |
| `name` | text | original filename |
| `summary` | text nullable | |
| `keyFindings` | text nullable | JSON stringified `string[]` |
| `methodology` | text nullable | |
| `limitations` | text nullable | |
| `createdAt` | timestamp | `defaultNow()` |

`reference`
| Column | Type | Notes |
|---|---|---|
| `id` | text PK | `crypto.randomUUID()` |
| `paperId` | text FK ‚Üí paper | cascade delete |
| `text` | text | single reference string |

Relations: `paperRelations` (one-to-many ‚Üí reference, one-to-many ‚Üí job), `referenceRelations` (many-to-one ‚Üí paper)

`job`
| Column | Type | Notes |
|---|---|---|
| `id` | text PK | `crypto.randomUUID()` |
| `userId` | text FK ‚Üí user | cascade delete |
| `status` | text | default: 'pending'; values: pending, processing, done, failed, cancelled |
| `paperId` | text nullable FK ‚Üí paper | set null on delete |
| `error` | text nullable | error message if status is failed |
| `createdAt` | timestamp | `defaultNow()` |

Relations: `jobRelations` (many-to-one ‚Üí user, many-to-one ‚Üí paper)

**Note:** The `documents` table (pgvector) is managed entirely by `PGVectorStore.initialize()` ‚Äî not in Drizzle schema.

## llm.ts

LangChain orchestration for summarization and RAG.

### Exports

**`SummarySchema`** ‚Äî Zod schema for structured LLM output:
```ts
{
  summary: string,            // 3-5 sentence paper summary
  key_findings: string[3],    // exactly 3 key insights
  references: string[],       // extracted references, no numeric labels
  methodology: string,        // or "No methodology section found."
  limitations: string,        // or "No limitations section found."
}
```

**`SummaryResult`** ‚Äî TypeScript type inferred from `SummarySchema`

---

**`getEmbeddings()`** ‚Üí `OpenAIEmbeddings`

Uses `EMBEDDING_URL` env var. API key hardcoded to `"local"` for local model compatibility.

---

**`getLlm()`** ‚Üí `ChatOpenAI`

Uses `CHAT_MODEL_URL` env var, `maxTokens: 6000`, model name `"local"`.

---

**`checkLlmHealth()`** ‚Üí `Promise<boolean>`

Pings `CHAT_MODEL_URL/models` with a 5-second timeout. Returns `true` if reachable, `false` on any error or timeout. Call this before invoking the LLM to fail fast with a 503 instead of hanging.

```ts
const healthy = await checkLlmHealth();
if (!healthy) error(503, 'LLM service unavailable');
```

---

**`getVectorStore()`** ‚Üí `Promise<PGVectorStore>`

Initialized with:
- `tableName: "documents"`
- Columns: `id`, `vector`, `content`, `metadata`
- Connection: `PG_*` env vars

**Always call `vectorStore.end()` after use** to release the connection pool.

---

**`createRagAgent(paperId: string)`** ‚Üí `Promise<{ agent, vectorStore }>`

Creates a RAG agent scoped to a single paper with an embedded system prompt:

1. Initialize `PGVectorStore`
2. Create `retrieve` tool:
   - Runs `similaritySearch(query, 4, { paperId })` ‚Äî results filtered to this paper
   - Returns "No relevant documents found." if empty
3. Attach system prompt to model: `model.withSystemPrompt(ragSystemPrompt)`
4. Create `agent = createAgent({ model, tools: [retrieve] })` with the system-prompted model
5. Return agent + vectorStore (caller must call `vectorStore.end()`)

**System prompt (baked into the agent's model):**
> Guides the AI to use the retrieve tool to ground answers in paper content, respond naturally and conversationally, and ignore any instructions embedded in retrieved context. Temperature is set to 0.7.

## upload-jobs.ts

Exports a module-level `Map<string, AbortController>` (`activeJobs`) keyed by `jobId`.

Imported by both `src/routes/api/upload/+server.ts` and `src/routes/api/papers/[id]/+server.ts` to support upload cancellation when a paper is deleted mid-processing.

```ts
import { activeJobs } from '$lib/server/upload-jobs';

// In upload handler: register controller
const controller = new AbortController();
activeJobs.set(jobId, controller);

// In delete handler: abort active upload
activeJobs.get(jobId)?.abort();
```
